# Latest Briefing: MetalSole

**Source:** Why Your AI-Improved Code Feels Wrong (2026-03-02)

## Key Points
- AI-driven code optimizations create "intent erosion" — thousands of small, functionally correct improvements that cumulatively strip purposeful design decisions from software
- Intent is invisible in codebases (not labeled, tested, or documented alongside code), making it the most vulnerable element as autonomous AI fixes accelerate
- Ship of Theseus problem: continuous micro-changes make products functionally better but emotionally flatter with no single breaking change to flag
- Example: western wear company's chatbot with intentional southern accent gets automatically removed when system optimizes for "professionalism"
- Erosion scales with volume of changes, not time; areas receiving most automated updates flatten first

## Notable Claims or Positions
- Core differentiator for engineers going forward is articulating what software should feel like and why — the human intent layer — not code-writing ability
- Claims this is "one of the most important undiscussed risks in software right now"
- Current development processes have "no mechanism to protect" intent: not tested, not stored alongside code, not audited

## Actionable Takeaways
- Encode intent directly in the codebase alongside code: document the why behind architectural choices, desired emotions, brand voice
- Place intent documents in the path of builders so anyone (human or AI) touching a domain must encounter them before making changes
- Lead with purpose rather than feature requirements when requesting work — the why guides hundreds of marginal implementation decisions
